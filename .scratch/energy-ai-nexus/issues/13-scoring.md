# 13 — Scoring + baselines + `/score` endpoint

## Parent

[PRD: Energy–AI Nexus Hackathon v1](../PRD.md)

## What to build

`scoring.py` exposes a pure function `score(world_final_state, baseline_p_ref, baseline_t_ref) -> dict` that computes the PRD's revised scoring formula:

```
P     = world.population
T     = world.treasury - STARTING_CASH
R     = world.cumulative_renewable_served_kwh / max(world.cumulative_total_served_kwh, 1)

p_term = 0.5 × min(P / P_ref, 3.0)              # cap at 3× per PRD
t_term = 0.4 × 0.5 × (1 + tanh(T / max(T_ref, 1)))
r_term = 0.1 × R

score  = p_term + t_term + r_term
```

The renewable-share accumulator runs as a per-hour pair of running sums in `world/economy.py`: `cumulative_renewable_served_kwh` and `cumulative_total_served_kwh`. **Curtailed kWh is excluded** from both numerator and denominator. All loads (residential, commercial, industrial, refinery, injection) count toward the denominator.

`GET /score` returns the score breakdown using the dev-seed baseline (committed to repo as `baselines/seed_42.json`). The format of `baselines/{seed}.json` is `{"p_ref": ..., "t_ref": ...}`. The baseline file for seed 42 is generated by slice 14 (scripted agent run).

## Acceptance criteria

- [ ] `scoring.score(world, p_ref, t_ref)` returns a dict with keys: `P`, `P_ref`, `p_term`, `T`, `T_ref`, `t_term`, `R`, `r_term`, `score`.
- [ ] `p_term` is capped at 0.5 × 3.0 = 1.5 even if `P >> P_ref`.
- [ ] `t_term` is in [0, 0.4]; very negative T saturates near 0; very positive T saturates near 0.4.
- [ ] `r_term` is in [0, 0.1].
- [ ] Renewable share accumulator: each hour, `cumulative_renewable_served_kwh += renewable_supply_kwh_this_hour` (excluding curtailed kWh), `cumulative_total_served_kwh += served_kwh_this_hour`.
- [ ] Curtailed kWh (the excess sold to the grid at export price) is excluded from both numerator and denominator.
- [ ] All load types (residential, commercial, industrial, refinery, injection) contribute to the denominator's `total_served_kwh`.
- [ ] `GET /score` returns the breakdown using the active baseline. If `baselines/seed_{N}.json` is missing for the active seed, the endpoint returns 404 with `"baseline_missing"`.
- [ ] Baseline file format is `{"seed": N, "p_ref": ..., "t_ref": ...}`.
- [ ] Tests in `tests/test_scoring.py` cover: p_term cap at 3×, t_term tanh saturation (very negative → ~0; very positive → ~0.4), r_term scaling, full integration with a synthetic final world.
